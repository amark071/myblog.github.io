Transformer基本架构详解

2025/11/15 大模型技术 LLM Transformer

Attention is all u need!

为什么选择Transformer？

  • 在此架构出现前，普遍的模型选取的架构还是RNN那样的循环神经网络结构：

    • 无法并行计算，效率很低（但可以加入teaching force解决）；

    • 长文本遗忘较为严重

    • 训练时会带来梯度爆炸等问题（可以加入残差连接）

  • Transformer架构提出了一种全新的自注意力机制，实现全局依赖的计算，并可非常轻松的实现并行化编程，大幅度提高高规模训练的可行性，很好的解决了RNN带来的问题。

  • 可以说没有Transformer，就没有今天的大模型。它不仅是技术底座，更是整个生成式AI浪潮的起点。

Decoder结构

  • decoder架构是transformer的解码器部分，其特点为掩码自注意力机制。

Embedding层

  • 最初我们尝试对token采用one-hot的方法转换成向量进行计算：
    • 我们将每一个词语都转换成一个长向量，其中每个向量都只有唯一的一个非零元1，其余均为0。
    • 不同的向量 eie_iei​和token一一对应。
    • 缺点：无法体现词与词之间的关系。
  • 于是提出了词嵌入的想法：
    • 将词语按照性质赋值，组合成一个向量：
      • 例如苹果是[1,0,0.7][1,0,0.7][1,0,0.7]，分别代表颜色、形状、甜度。这样相同性质的词语距离会接近。
词嵌入的流程与技术
原始句子
   ↓ ① 分词（Tokenize）
list_of_tokens = ["ChatGPT", "是", "一款", "大型", "语言", "模型"]
   ↓ ② 映射到词典 ID
token_ids = [24743, 318, 1021, 2134, 4287, 4568]
   ↓ ③ 词嵌入（Embedding 层）
h0 = E[token_ids]  ← 每一行 768/1024/4096 维向量
   ↓ ④ 加位置编码，送入 Transformer

  1. 分词（tokenization）：先将输入的文本切分成一块一块的token，以便转向量。

  2. 编码（可以用one-hot）：按照每个token的进行one-hot编码，组合成一个输入矩阵。

  3. 词嵌入（embedding）：将输入矩阵和词嵌入字典矩阵相乘，得到词嵌入后的输入矩阵。

  • 输入矩阵：token数×\times×字典词汇量数（即embedding权重矩阵一共可以编码多少个token）。

  • 嵌入矩阵：字典词汇量数 ×\times×嵌入维度（嵌入后每个token的大小）。

  1. 位置嵌入（positon embedding）：将每个词语加上自己的位置向量。
  • 正-余弦绝对位置：
    • 对于位于句子中第pos位的token，我们对其进行以下位置编码：
PE(pos,2i)   = sin(pos / 10000^(2i/d))
PE(pos,2i+1) = cos(pos / 10000^(2i/d))
# 2i/2i+1表示向量维度的下标
# 实现时可以提前准备好矩阵，然后数pos按行加上去就行了
  • 可学习绝对位置：直接把上述的位置矩阵变成一个可学习的参数矩阵就行了。

Attention层

  • 注意到在谈论decoder的时候一般是这这个步骤而不是包含前面和后面的所有。

  • Embedding层中，我们输入了文本，转换成了token数(s)×\times×字典词汇量数(v)，tokens数(s) ×\times×嵌入维度(d)的矩阵X，进入到attention层中。

  • 注意到Attention部分一般有l层，即会将我们下面讲解的部分重复进行，而每一层都利用上一层的输出作为输入。

Masked Multi-Head Self-Attention（带掩码的多头自注意力）
  • 首先利用三个可学习的矩阵WQW_QWQ​，WKW_KWK​，WVW_VWV​，将输入矩阵X投影层三个矩阵QQQ，KKK，VVV，分别代表：“查询矩阵”、“键矩阵”、“值矩阵“。
  • 然后将其按头拆分（多头注意力，直接按列等分成a份）。
  • 对每个头计算自注意力分数矩阵：
    • 意味着通过查询矩阵和键矩阵的相似程度算出一个概率分布，然后加权到值矩阵上作为注意力得分，通过dkd_kdk​（即每个头的列数）控制大小。
Attention(Q,K,V)=softmax(QKTdk)V \text{Attention}(Q,K,V)=\text{softmax}(\frac{QK^T}{\sqrt{d_k}})V Attention(Q,K,V)=softmax(dk​​QKT​)V
  • 注意到：在上述的过程中，当我们求得自注意力后，需要做上三角掩码（即用类似下面的矩阵加到自注意力分数矩阵上）
    • 只保留自己和自己之前token的注意力，防偷窥后面的token
    • 这样之后softmax之后，权重矩阵就会变成一个下三角矩阵
       0   1   2   3
    0  0  -∞  -∞  -∞
    1  0   0  -∞  -∞
    2  0   0   0  -∞
    3  0   0   0   0
  • 最后我们再将所有头按分割顺序拼成一个大型矩阵AAA。
  • 对矩阵AAA做一次线性全连接，混合各个头的训练结果，也可以加偏置。
  • 然后我们需要对矩阵做一个残差链接并标准化处理：
SubLayerOut = LayerNorm(x + Dropout(SubLayer(x)))
  • Dropout：训练时随机置 0 部分元素，防止 co-adaptation；推理时全部保留。设丢弃概率ppp（原论文 p=0.1p=0.1p=0.1）生成同样形状的掩码矩阵M∼Bernoulli(1−p)M\sim Bernoulli(1-p)M∼Bernoulli(1−p)，然后将其与原矩阵逐分量相乘之后处以1−p1-p1−p。
    • 这样得到的数据方差增大但期望不会变。
    • 意义：每次前向传播时随机丢弃某些激活，可以强迫网络不依赖单个特征。
  • LayerNorm：计算每一行的均值μ\muμ和方差σ2\sigma^2σ2，对每行的元素做标准化z^=z−μσ2+ϵ\hat{z}=\frac{z-\mu}{\sqrt{\sigma^2+\epsilon}}z^=σ2+ϵ​z−μ​。
    • 可以再做一个仿射变换LN(z)=γz^+βLN(z)=\gamma\hat{z}+\betaLN(z)=γz^+β，并初始化为γ=1,β=0\gamma=1,\beta=0γ=1,β=0。恢复归一化可能破坏的表达能力，让网络自己更新参数决定要保持哪种尺度。
  • 残差连接的数学意义： 避免梯度消失导致过深的网络反向传播失败。并且有工作显示此举可以平滑曲面，加速收敛，提高鲁棒性。
  • 注意到：GPT等论文会把标准化做到残差之前，即输入时候标准化。类似于：
SubLayerOut = x + Dropout(SubLayer(LayerNorm(x))
Cross Attention（与Encoder层的信息交流层）
  • 注意到：如果是类似GPT这样的Decoder-only的架构，这层就变成了没有掩码的自注意力层，其将完全类似于上述流程，仅仅只是少了加掩码这一流程。

  • Cross Attention 的独特之处在于，QQQ和 KKK、VVV 的来源不同。比如：

    • QQQ 来自序列 AAA（比如目标语言的单词）。
    • KKK 和 VVV来自序列 BBB（比如源语言的句子）。

    • 这样，序列 AAA就能通过“询问”序列 BBB来获取相关信息。

  • 在Transformer的结构中，AAA是decoder层，BBB是encoder层。除此以外的步骤则和不加掩码的attention完全一致。

Feed Forward（前馈层）
  • FFN 是 Transformer 块里唯一不带时间维度的子层，作用等价于“对每个 token 单独做一次两级 MLP 非线性变换”，给模型加局部内插能力。
FFN(X)=ReLU(XW1+b1)W2+b2 FFN(X)=ReLU(XW_1+b_1)W_2+b2 FFN(X)=ReLU(XW1​+b1​)W2​+b2
  • 一般来说：W1W_1W1​的维度为：d×4dd\times 4dd×4d,W2W_2W2​的维度为：4d×d4d\times d4d×d，最后保证了输出的维度仍然为：s×ds\times ds×d。

线性层与输出头

  • 经过lll层的decoder层次的注意力训练，我们最后只需将所得到的矩阵的维度转换成合适的维度：
    • 此时应该转化为s×vs\times vs×v，因为输出时是希望从所有的词汇表中选取。
  • 最后我们做一次softmax求出下一个词的概率分布，并作出预测即完成了整个流程。

Encoder结构

  • 可以从图中看到，encoder结构和decoder有几分相似，最大的不同点在于：在做attention的时候，我们没有添加掩码。这也体现出了encoder和decoder架构最大的区别：

    • decoder是基于当前词语和之前出现过的词语进行的；
    • encoder是基于全局词语进行的。

  • 整体训练流程：

    • Transformer 的训练阶段用的是并行教师强制策略（teacher forcing）：
      1. Encoder 先运行，产出记忆矩阵 M；
      2. 紧接着 Decoder 在一次并行计算里同时看全部目标序列（右移一位 + mask），并去 M 上做 Cross-Attention；
      3. 损失只比较 Decoder 输出的 T 个位置与T 个真值 token；
      4. 反向传播时梯度既回传 Decoder 也回传 Encoder，整套参数同步更新。

decoder-only的参数分析

  • 参数分析是一个很有意义的步骤，可以有效的帮助我们分析整个流程的性能受限于何。

Parameters——模型参数

  • 词嵌入层：需要训练词嵌入权重矩阵，peb=vdp_{eb}=vdpeb​=vd；
  • attention层：
    • 分别训练QKVQKVQKV，并训练最后的线性全连接层，共3d2+d2=4d23d^2+d^2=4d^23d2+d2=4d2；
    • 前馈层：训练两个全连接层矩阵，4d2∗2=8d24d^2*2=8d^24d2∗2=8d2；
    • 所以attention层共需要12ld212ld^212ld2参数。
  • 线性输出层：同词嵌入层的大小，vdvdvd。
  • 于是在我们忽略到偏置的大小后，我们需要训练的参数大小为：2vd+12ld22vd+12ld^22vd+12ld2。

Activations——模型激活值（理解为中间量）

  • 词嵌入层：需要存储XXX，大小为sdsdsd。

  • attention层（lll层，aaa头）：

    • 需要分别存储QiQ_iQi​、KiK_iKi​、ViV_iVi​，大小为3sd3sd3sd；

    • 计算自注意力分数时，存储QiKiTQ_iK_i^TQi​KiT​、以及其softmax值和加权值矩阵，共2s2a+sd2s^2a+sd2s2a+sd；

    • 存储全连接后的矩阵，大小为sdsdsd；

    • 存储前馈层的中间结果：4sd+4sd+sd4sd+4sd+sd4sd+4sd+sd，最后一项是指存储激活后的结果。

    • 所以attention层共需要存储(2s2a+14sd)l(2s^2a+14sd)l(2s2a+14sd)l激活值。

  • 线性层+softmax：sv+svsv+svsv+sv。

  • 最后考虑bbb个batch，忽略掉2sv2sv2sv，大致的激活值数量为：(2s2a+14sd)lb(2s^2a+14sd)lb(2s2a+14sd)lb

内存消耗

  • 我们有以下对内存的估计：
Memory = model parameters + gradient + optimizer states + activations
  • 其中gradient大小和model parameters相仿，optimizer states若是采用的优化器为：Adam，则需要存储一阶和二阶动量，大约为model parameters的两倍大小：
  • 总的memory消耗估计为：48ld2+bl(2s2a+14sd)×4Bytes48ld^2+bl(2s^2a+14sd)\times 4Bytes48ld2+bl(2s2a+14sd)×4Bytes

计算量估计

  • 我们首先注意到一个前提：对于大小为m×nm\times nm×n和n×pn\times pn×p的矩阵相乘，计算量为：2mnp2mnp2mnp。

  • 词嵌入层：计算量为2svd2svd2svd。

  • attention层（lll层，aaa头）：

    • 需要分别计算QiQ_iQi​、KiK_iKi​、ViV_iVi​，大小为6sd26sd^26sd2；

    • 计算自注意力分数时，共4s2d4s^2d4s2d；

    • 存储全连接后的矩阵，计算量为2sd22sd^22sd2；

    • 计算前馈层：8sd2+8sd28sd^2+8sd^28sd2+8sd2。

    • 所以attention层共需要的计算量为：(24sd2+4s2d)l(24sd^2+4s^2d)l(24sd2+4s2d)l。

  • 线性层：2sdv2sdv2sdv。

  • 综上所述，考虑bbb个batch，前向传播总共需要的计算量为bl(24sd2+4s2d)+4bsvdbl(24sd^2+4s^2d)+4bsvdbl(24sd2+4s2d)+4bsvd

  • 考虑到反向传播的计算量约等于前向的两倍，故总计算量约为：
